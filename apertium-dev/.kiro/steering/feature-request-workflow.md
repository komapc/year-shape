# Feature Request & Bug Fix Workflow

When you receive a feature request or bug fix request, follow this structured workflow (unless explicitly stated "DO NOW"):

## 1. Clarification Phase
Ask 1-3 targeted questions to ensure you fully understand the requirements and scope of the request.

## 2. Solution Planning
Clearly explain your proposed approach, including:
- What changes will be made
- Which files will be affected
- Any potential risks or considerations

## 3. Implementation
Once approved, implement the requested changes following best practices and project conventions.

## 4. Testing & Validation
When applicable:
- Add appropriate test cases
- Run existing tests to ensure no regressions
- Validate the implementation works as expected

## 5. Pull Request Creation
- Create a feature branch
- Make atomic commits with descriptive messages
- Create a pull request with clear description
- Provide the PR link to the user for review
